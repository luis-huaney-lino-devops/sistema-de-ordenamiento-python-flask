Teoría del Quick Sort

Quick Sort es un algoritmo de ordenación eficiente de tipo divide y vencerás que selecciona un elemento como pivote y particiona la lista en dos sublistas: una con elementos menores que el pivote y otra con elementos mayores o iguales que el pivote.

Proceso del Quick Sort

  • Selecciona un elemento como pivote (puede ser el primer elemento, el último, el central, o uno aleatorio).
  • Reordena la lista de manera que todos los elementos menores que el pivote queden a su izquierda y todos los elementos mayores o iguales que el pivote queden a su derecha.
  • Aplica recursivamente el mismo proceso a las dos sublistas generadas.

Características del Quick Sort

  • Es uno de los algoritmos de ordenación más rápidos en promedio para listas grandes.
  • Requiere menos memoria que Merge Sort porque no necesita espacio adicional para la combinación de sublistas.
  • No es un algoritmo estable, lo que significa que no necesariamente mantiene el orden relativo de los elementos con valores iguales.
  • Funciona bien en memoria caché debido a su acceso secuencial a los datos.

Complejidad del Quick Sort

  • Peor caso: O(n²) (ocurre cuando el pivote elegido es el menor o el mayor elemento en repetidas ocasiones)
  • Mejor caso: O(n log n) (ocurre cuando el pivote divide la lista en dos sublistas casi iguales en cada partición)
  • Promedio: O(n log n)

Algoritmo del Quick Sort

def quick_sort(arr):
  if len(arr) <= 1:
    return arr
  else:
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quick_sort(left) + middle + quick_sort(right)
